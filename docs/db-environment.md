# db-environment.md

## DB 환경 구성

### 1) 개발 환경 구성 방식
- **MS-SQL을 Docker로 실행**해서 DB 버전을 고정하고, 실행 환경을 동일하게 유지했습니다.
- 로컬 OS에 DB를 직접 설치하지 않고 컨테이너로 올려서, 프로젝트 시작/중지/재구성이 쉽습니다.
- 환경 초기화가 필요할 때도 컨테이너 단위로 정리할 수 있어 관리가 단순합니다.

### 2) DBeaver 사용 이유
- **DB 접속 및 데이터 확인**: 테이블 구조, 인덱스, 데이터 상태를 빠르게 확인할 수 있습니다.
- **쿼리 실행 및 관리**: SQL 실행, 결과 확인, 저장한 쿼리 재사용이 편해서 개발/디버깅 속도가 좋습니다.

### 3) 전체 구조
- Spring Boot 백엔드가 Docker로 실행 중인 MS-SQL에 JDBC로 연결합니다.
- DBeaver는 같은 MS-SQL에 접속해 스키마/데이터를 조회하고 쿼리를 실행합니다.

```text
[Spring Boot Backend]
          |
          v
   [MS-SQL (Docker)]
          ^
          |
      [DBeaver]
```

### 4) 사용 방식 요약
- 애플리케이션: Spring Boot에서 DB 트랜잭션 처리
- 데이터 확인/운영: DBeaver에서 조회 및 쿼리 점검
- 인프라: Docker Compose로 DB 실행/중지 관리
